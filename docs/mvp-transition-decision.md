# MVP Transition Decision

This note is the Phase 9 MB-05 handoff decision for issue #228. It uses the
P9 acceptance report regenerated by:

```bash
python3 scripts/evaluate_dataset.py --poc-acceptance-report
```

The report is generated from `datasets/poc_evaluation_manifest_v1.json`,
`datasets/gold/llm_stability_runs_v0.json`, and
`datasets/gold/poc_mode_comparison_v1.json`. The regenerated clean-worktree
report keeps the MVP-before result at `fail`, with 4 failed, 4 passed, and 0
unknown criteria. The acceptance blockers remain the functional,
structured-output, LLM-control, and log gates. The report also states that it
is not the final MVP go/no-go decision and is not formal GMP validation.

## Recommendation

MVP planning may continue, but MVP implementation should not start as a
production-readiness track until the MVP-before gates below are closed.

The PoC has enough evidence to plan a constrained MVP around synthetic,
review-first conversion workflows. It does not yet justify claiming GMP
production use, confidential-record handling, or unattended conversion output.
The MVP scope should remain fail-closed: source-linked review artifacts are
allowed, while any missing provenance, missing audit evidence, unstable LLM
behavior, or failed representative conversion row blocks promotion.

## MVP-before conditions

- Re-run `python3 scripts/evaluate_dataset.py --poc-acceptance-report` and
  require `overall_status: pass` before treating Phase 9 as implementation
  ready.
- Resolve all representative conversion harness failures. The regenerated
  harness records 12 of 16 representative conversion rows completing and 4
  rows still failing or failing closed. Those rows remain explicit MVP-before
  gate blockers, not successful artifact output.
- Require primary artifact structure/source expectations to pass for the target
  MVP modes. Current Word/Excel representative artifact expectations pass;
  `pdf_to_word` and record-PDF rows now produce audited conversion evidence in
  the local evaluation environment. `pdf_to_excel` still fails artifact
  expectations for the table-report fixture, and `scanned_pdf_ocr` remains
  blocked until a public synthetic or anonymized scanned/OCR fixture is added.
- Require audit evidence for every P9 harness outcome. Fail-closed dependency
  or fixture-gate rows are not promotable until they either produce normal
  audited conversion evidence or remain documented as explicit MVP-before
  blockers.
- Enforce the LLM stability acceptance threshold. The current report fails LLM
  correction/completion because the synthetic runs have 2 unstable examples,
  plan agreement below 1.0, confirmed-value agreement below 1.0, schema
  failures above 0.0, and deterministic fallback above 0.0. External AI API
  guard violations remain at 0 and are reported separately in the same LLM
  control evidence.
- Preserve authenticated PoC API behavior before treating security as accepted.
  The current report records zero external AI API guard violations and tracked
  authenticated PoC API session coverage, including an env-backed local token
  success path.

## Remaining work

MVP should preserve the PoC guardrails and remove ambiguity before expansion:

- Keep external AI API blocking and high-risk false auto-confirmation guards in
  place.
- Keep review-first output semantics. Source linkage passed in the report, but
  failed artifact structure and missing audit rows mean outputs must not be
  treated as normal unattended results.
- Make audit evidence a hard boundary for promotion and restore paths.
- Convert the report's follow-up candidates into small issues before starting
  broader MVP work.

## GMP record PDF handling

GMP record PDFs should be included in MVP planning only as public synthetic or
anonymized record-PDF-equivalent fixtures. They should not be included as real
GMP production records in the initial MVP.

Before real GMP record PDFs can enter scope, VeriDoc needs a separate validation
track that covers formal QA approval, confidential-data controls, source-record
retention, reviewer accountability, audit trail completeness, and regulatory
sign-off. Until then, any GMP PDF output must be labeled as review evidence,
not as source replacement or validated production output.

`docs/gmp08-acceptance-evaluation.md` remains the GMP acceptance reference for
public synthetic record-PDF-equivalent evidence. Its eight GMP-08 criteria pass
for the synthetic record dataset, but that is not the same as approving real
GMP production use.
